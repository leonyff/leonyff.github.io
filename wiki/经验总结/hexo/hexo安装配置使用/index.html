hexo安装配置使用

hexo安装

hexo依赖于：Node.js 和 Git

安装命令

初始化

其它命令

init

新建一个网站，如果没有设置[folder]，Hexo默认在目前的文件夹建立网站

new

新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。

参数：

  • -p, --path 自定义新文章的路径
  • -r, --replace 如果存在同名文章，将其替换
  • -s, --slug 文章的slug， 作为新文章的文件名和发布后的URL

以上命令会创建一个 source/about/me.md 文件，同时 Front Matter 中的 title 为 "About me"

注意！title 是必须指定的！如果你这么做并不能达到你的目的

generate

生成静态文件

参数：

  • -d, --deploy 文件生成后立即部署网站
  • -w, --watch 监视文件变动
  • -b, --bail 生成过程中如果发生任何未处理的异常抛出异常
  • -f, --force 强制重新生成文件

publish

发表草稿

server

启动服务器。默认情况下，访问网址为： http://localhost:4000/

参数：

  • -p, --port 重设端口
  • -s, --static 只使用静态文件
  • -l, --log 启动日记记录，使用覆盖记录格式

deploy

部署网站

参数：

-g, --generate 部署之前预先生成静态文件

目录结构

创建完成目录结构如下：

_config.yml

网站的配置文件，这个文件可以配置网站的大部分参数

package.json

配置网站的应用程序信息，可以自由移除。扩展网站功能。

scaffolds

模板文件夹，新建文章时，Hexo会根据scaffold来建立文件

Hexo的模板指的是在新建文章文件时，文章中默认填充的内容。

source

资源文件夹时存放用户资源的地方，除了_posts文件夹外，开头命名为_的文件/文件夹和隐藏文件会被忽略。

Markdown和Html文件会被解析并存放到public文件夹，而其它文件会拷贝过去。

themes

主题文件夹，Hexo会根据主题来生成静态页面

配置 config.yml

网站：site

参数 描述
title 网站标题
subtitile 网站副标题
description 网站描述
keywords 网站关键字，多个用半角逗号分隔
author 作者
language 网站使用的语言：zh-Hans
timezone 区域：Asiz/shanghai

网址：url

参数 描述 默认值
url 网址
root 网站根目录
permalink 文章永久链接格式 :year/:month/:day:title/
permalkink_defaults 永久链接中各部分的默认值

如果网站存放在子目录中，例如：http://domain.com/blog, 要将url设为: http://domain.com/blog，并把root设置为/blog/

目录：Directory

参数 描述 默认值
source_dir 资源文件夹，这个文件夹用来存放内容 source
public_dir 公共文件夹，这个文件夹用于存放生成的站点文件 public
tag_dir 标签文件夹 tags
archive_dir 归档文件夹 archives
category_dir 分类文件夹 categories
code_dir include code文件夹，source_dir下的子目录 downloads/code
i18n_dir 国际化（i18n）文件夹 :lang
skip_render 跳过指定文件的渲染，匹配到的文件将会不做改动的复制到public目录中，可以使用glob表达式来匹配路径

文章：writing

参数 描述 默认值
new_post_name 新的文件名称 :title.md
default_layout 预设布局 post
auto_spacing 在中文和英文之间加入空格 false
titlecase 把标题转换为title case false
external_link 在新标签打开链接 true
filename_case 把文件名称转换为小写(1)或大写(2) 0
render_drafts 显示草稿 false
post_asset_folder 启动Asset文件夹 false
relative_link 把链接改为于更目录相对地址 false
future 显示未来的文章 true
highlight 代码块的设置

分类&标签：Category & Tag

参数 描述 默认值
default_category 默认分类 uncategorized
category_map 分类别名
tag_map 标签别名

日期/时间格式：Date / Time format

参数 描述 默认值
date_format 日期格式 YYYY-MM-DD
time_format 时间格式 HH:mm:ss

分页：Pagination

参数 描述 默认值
per_page 每页显示文章的数量(0: 关闭分页功能) 10
pagination_dir 分页目录 page

扩展：Extensions

参数 描述
theme 当前主题名称，值为: false 时禁用主题
theme_config 主题配置文件，这里放置的配置会覆盖，主题目录下的config.yml中的配置
deploy 部署部分的设置
meta_generator Meta generator标签，值为false时Hexo不会再头部插入该标签
